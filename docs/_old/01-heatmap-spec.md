# TwitCasting Livefield / Heatmap 仕様確定

## 1. この機能の役割

Heatmap は、**今この瞬間のツイキャス配信空間の勢力図**を見る機能とする。

この画面で分かることは3つだけに固定する。

* **誰が大きいか**
  = 現在視聴者数が多い配信
* **誰がざわついているか**
  = 視聴者規模に対してコメント速度が高い配信
* **誰が今伸びているか**
  = 直近数分で視聴者数が増えている配信

Heatmap は「比較表」ではなく、**ライブ空間の瞬間的な熱分布を直感で掴む画面**とする。

---

## 2. 1ノードの定義

Heatmap 上の1つのノードは、**その時点で配信中の1ライブ枠**とする。
配信者単位ではなく、**現在配信中のライブセッション単位**で扱う。

つまり、

* 同じ配信者が今ライブ中なら1ノード
* 配信終了したらノード消滅
* 次回配信開始時は新しいライブ枠として再登場

で固定する。

---

## 3. 取得・更新周期

V1では **1分ごと更新** で固定する。

### 収集周期

* サーバー側収集: 1分ごと
* フロント表示更新: 新しいスナップショットを受け取るごと
* 自動更新: デフォルトON
* 手動Refresh: あり

### 理由

* コメント速度を見るには5分更新だと鈍い
* 30秒更新は無料運営前提だとやりすぎ
* 1分なら「今感」と「運営コスト」のバランスがいい

---

## 4. 表示対象

Heatmap に出す対象は、**その時点で配信中の上位ライブ枠**とする。

### 初期表示

* デフォルト: **Top 50**
* 切替:

  * Top 20
  * Top 50
  * Top 100

### 上位判定基準

並びの母集団は **現在視聴者数順** とする。
つまり表示対象に入るかどうかは、まず viewers で決める。

### その他

* 非ライブ枠は表示しない
* 明らかに欠損したデータは除外
* 同一枠の重複は許可しない

---

## 5. Heatmap の見た目ルール

Heatmap の中心は **Canvas 上の bubble map** に固定する。

各ノードの見た目は次で決める。

### 5-1. ノードの大きさ

**大きさ = 現在視聴者数**

ただし線形ではなく、半径は平方根圧縮を使う。

例:
`radius = base + scale * sqrt(viewers)`

これにより、

* 大手だけが巨大化しすぎない
* 中規模も見える
* 小規模も消えない

### 5-2. ノードの色

**色の濃さ = viewers の相対順位**

* 上位ほど濃い
* 下位ほど薄い
* 色相は基本1系統で固定
* 多色化しない

つまり色はカテゴリ分けではなく、**存在感の強弱**だけに使う。

### 5-3. ノードの震え

**震え = コメント盛り上がり**

ただし総コメント数ではなく、以下の仮式を使う。

`cpm = Δcomments / Δminutes`
`agitation_raw = cpm / (viewers + 20)`

この `agitation_raw` をその瞬間の表示対象内で相対評価し、
**Lv0〜Lv5** に落とす。

### 5-4. ノードの発光

**発光 = 直近の伸び**

伸びは v1 では以下で定義する。

`momentum_raw = (viewers_now - viewers_prev) / max(viewers_prev, 20)`

* 前回より伸びていると発光
* 横ばいは弱い
* 減少は発光なし

発光はリングか薄い外周グローで表現する。
ノード本体の色は変えない。

---

## 6. コメント盛り上がりの段階

震えは **6段階表示** に固定する。

* Lv0: 静止
* Lv1: 微振動
* Lv2: 弱い震え
* Lv3: はっきり震える
* Lv4: 強い震え
* Lv5: 激しく震える

### 段階化ルール

固定閾値ではなく、**その瞬間のサイト内相対分布**で切る。

初期ルール:

* 下位30% → Lv0
* 30–50% → Lv1
* 50–70% → Lv2
* 70–85% → Lv3
* 85–95% → Lv4
* 上位5% → Lv5

これで昼夜やイベント日の差に耐える。

### 補足

この式と閾値は **V1仮採用**。
raw データを残す前提で、実データ観測後に調整可能とする。

---

## 7. ノード配置

配置は「見た目重視のランダム」ではなく、**安定して読める配置**にする。

### 配置原則

* viewers 上位ほど中央寄り
* 下位ほど外周寄り
* 毎更新で大移動しない
* 同一枠は前回位置から滑らかに補間
* ノード同士は重なりすぎない

### v1の配置方式

v1 は複雑な force simulation を主役にしない。
基本は **安定配置 + 軽い衝突回避** でよい。

### 新規登場

新規ノードは外周からフェードインして定位置へ入る。

### 配信終了

終了ノードは即消しせず、

* 透明度低下
* 少し縮小
* フェードアウト
  してから消す。

---

## 8. アニメーションの規則

Heatmap は動きが大事だが、うるさくしない。

### 許可する動き

* agitation に応じた震え
* momentum に応じた発光
* 更新時の位置補間
* hover/tap 時の軽い拡大

### 禁止する動き

* 常時激しい飛び跳ね
* 毎回の全面再レイアウト
* 粒子の大量散布
* 背景の常時流体演出

### animation toggle

ユーザーは Animation On/Off を切り替え可能にする。

* On: 通常表現
* Off: 震え停止、発光弱化、位置補間のみ

## 9. 画面UI構成

Heatmap ページの構成は以下で固定する。

### 上部

* ページ名
* 短い説明
* Last updated
* Coverage
* Status

### コントロール

* Top 20 / 50 / 100
* Comment window 1m / 3m / 5m
* Animation On/Off
* Auto refresh On/Off
* Refresh

### 本体

* Heatmap canvas

### 補助

* 凡例
* summary metrics
* 詳細パネル

### 詳細パネル

ノード選択時に以下を表示する。

* streamer name
* current title
* current viewers
* comments/min
* agitation level
* momentum
* current rank by viewers
* current rank by agitation
* current rank by momentum
* stream link

---

## 10. Summary metrics

Heatmap 上部または直下に4つだけ出す。

* Active streams
* Total viewers observed
* Highest agitation stream
* Strongest momentum stream

ここでは数字を詰め込みすぎない。

---

## 11. ラベル表示

全ノードに常時ラベルは出さない。

### 常時表示

* Top 10: 名前 + viewers
* 11〜20位: 名前のみ
* それ以下: 非表示

### hover / tap 時

* フル情報表示
* 選択枠を強調
* 他ノードは少し弱める

---

## 12. モバイル挙動

モバイルでは見せ方を簡略化する。

### 残すもの

* Top 20 / 50
* 1m / 3m
* Refresh
* Animation On/Off
* ノードタップ詳細

### 削るもの

* Top 100
* 細かいラベル密度設定
* 複雑な表示モード切替
* 長い説明文

### 詳細表示

モバイルでは詳細は **bottom sheet** で出す。

---

## 13. データ保存

後で式調整できるよう、raw データは保存する。

最低保存項目はこれ。

* timestamp
* streamer_id
* movie_id
* streamer_name
* title
* viewers
* comment_count
* delta_comments
* cpm
* agitation_raw
* agitation_level
* momentum_raw

保存先の役割は以下。

* **D1**: 履歴保存
* **KV**: 最新表示用の軽量キャッシュ

---

## 14. 状態設計

Heatmap は以下の状態を持つ。

### loading

* 骨組み表示
* `Loading live field...`

### live

* 通常表示
* last updated が現在に近い

### stale

* 最終更新が遅延
* 黄色 note を表示
* 最後の正常状態を維持

### empty

* 表示条件に一致する配信なし
* reset / widen controls を促す

### error

* 赤 note
* retry ボタン
* demo mode 切替可能

### demo

* デモデータで仮表示
* 本番と明確に区別

---

## 15. Heatmap がやらないこと

v1 の Heatmap では以下をやらない。

* クロスサイト比較
* ジャンル別クラスタ
* フォロー配信者優先表示
* ログイン保存
* 配信者プロフィール分析
* コメント本文解析
* Battle Lines 的な時系列比較

Heatmap はあくまで **瞬間観測** に限定する。

---

## 16. v1 完成条件

Heatmap の完成条件はこれで固定する。

### 必須

* 現在配信中の Top N を表示できる
* 視聴者数でノードサイズが変わる
* agitation Lv0〜Lv5 が見た目に反映される
* momentum が発光で見える
* 1分更新で滑らかに反映される
* ノード選択で詳細が見られる
* stale / empty / error / demo が分かる

### 不可

* 更新のたびに配置が崩れる
* 小規模配信が agitation だけで主役化しすぎる
* モバイルで操作不能
* 何を見ているか説明なしで分からない

---

## 17. この機能の一言定義

最後に一文で固定する。

**Heatmap は、TwitCasting 内で今どの配信が大きく、騒がしく、伸びているかを、点の勢力図として瞬時に読む機能。**
