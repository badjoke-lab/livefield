# TwitCasting Livefield 共通仕様書 v1.0

## 1. 文書の位置づけ

本書は、TwitCasting Livefield 全体に適用する**共通ルール**を定める。
各機能仕様に記載された内容であっても、本書に反する場合は本書を優先する。

対象機能は以下の3つとする。

* Heatmap
* Day Flow
* Battle Lines

---

## 2. サイト全体の目的

TwitCasting Livefield は、TwitCasting 内の配信状況を **「今の勢力」「1日の支配率」「個別比較」** の3方向から可視化する観測サイトとする。

ただし本サイトは、見た目の派手さを競うサイトではない。
最重要目的は以下とする。

1. **ブラウザが安定動作すること**
2. **何を見せる画面かが一瞬で分かること**
3. **その上で、最低限の演出で熱量を伝えること**

---

## 3. 全体優先順位

TwitCasting Livefield における優先順位は、全ページ・全機能で以下の順に固定する。

1. **安定動作**
2. **可読性**
3. **更新の滑らかさ**
4. **演出**
5. **追加機能の多さ**

明文化すると、以下の一文を全体原則とする。

**Livefield は、どの機能でもブラウザの安定動作を最優先とし、演出は描画予算の範囲内でのみ許可される。**

---

## 4. 対象範囲

本書が規定する対象は以下とする。

* サイト全体の技術構成
* データ取得周期
* クライアント描画方式
* 画面更新ルール
* 表示件数上限
* アニメーション上限
* モバイル制限
* フェイルセーフ
* 禁止事項

---

## 5. 非対象

本書では以下を扱わない。

* TwitCasting API の詳細な各エンドポイント定義
* 個別画面の色や文言の最終調整
* 将来のクロスサイト比較仕様
* ログインやユーザー保存機能
* 収益化機能
* AI要約機能
* コメント本文解析

---

## 6. 全体技術構成

TwitCasting Livefield の v1 構成は以下で固定する。

### 6.1 配信

* **Cloudflare Pages**
* 静的ファイル配信を基本とする

### 6.2 定期収集

* **Cloudflare Workers + Cron**
* 1分ごとに定期収集する

### 6.3 保存

* **D1**: raw / rollup / 日別集計の本体保存
* **KV**: 最新表示用の軽量キャッシュのみ

### 6.4 フロントエンド

* **plain TypeScript を基本**
* 描画コアは **framework-independent** に実装する
* React / Next.js を使う場合でも、**描画ループの主役にしない**
* React state / Next state で毎フレーム描画を回す設計は禁止

### 6.5 描画方式

* **Canvas 2D 主体**
* DOM / SVG は UI 部分に限定
* WebGL は v1 では採用しない
* WebGL 化は、Canvas 2D での予算内運用が破綻した場合のみ再検討する

---

## 7. 基本データ更新ルール

### 7.1 収集周期

* サーバー側収集: **1分ごと**
* Heatmap: 1分更新表示
* Day Flow: 1分収集を 5分バケット中心で表示
* Battle Lines: 1分 / 5分 / 10分 の切替可

### 7.2 当日データ

* 当日ページは最新スナップショットを反映する
* 現在時刻以降の未来区間は表示しない

### 7.3 過去日データ

* 過去日は固定データとして扱う
* ページ表示のたびに再集計しない

### 7.4 stale 判定

最終正常更新から一定時間以上遅れた場合、画面を stale とする。

初期ルール:

* **3分以上遅延**: stale note 表示
* **10分以上遅延**: strong stale としてより強く表示

### 7.5 partial 判定

* 一部バケット欠損
* 一部配信者のみ欠損
* 集計途中

のいずれかがあれば partial 状態として扱う。

---

## 8. クライアント描画の基本方針

### 8.1 描画の主役

* Heatmap / Day Flow / Battle Lines の主描画は **Canvas 2D**
* UI操作、ボタン、凡例、パネルのみ DOM で持つ

### 8.2 再描画

* 再描画は **requestAnimationFrame** を基準に行う
* データ更新のたびに無制限に描画しない
* 新規スナップショット到着時のみ状態更新し、アニメーションは必要最小限で補間する

### 8.3 高解像度制御

Canvas の内部解像度は無制限に上げない。

共通ルール:

* `devicePixelRatio` は **上限 2** まで
* 低負荷モードでは **上限 1.5** まで
* モバイルでは必要に応じて **1** に落としてよい

### 8.4 非表示時の動作

* タブ非表示時はアニメーションを停止または極小化する
* 背景タブでの常時描画は禁止
* 再表示時に必要な分だけ再同期する

---

## 9. 表示件数・描画予算の上限

これを超える表示は **仕様違反** とする。

### 9.1 Heatmap

* デフォルト: **Top 50**
* デスクトップ最大: **Top 100**
* タブレット最大: **Top 50**
* モバイル最大: **Top 50**
* 常時ラベル表示: **Top 10 まで**
* 強調マーカー常時表示: **上位数件のみ**

### 9.2 Day Flow

* デフォルト: **Top 20 + Others**
* デスクトップ最大: **Top 50 + Others**
* タブレット最大: **Top 20 + Others**
* モバイル最大: **Top 20 + Others**
* 常時ラベル表示: **Top 5〜10**
* 常時イベントマーカー: **Peak / Heat を必要最小限**

### 9.3 Battle Lines

* デフォルト: **Top 5**
* デスクトップ最大: **Top 10**
* タブレット最大: **Top 5**
* モバイル最大: **Top 5**
* 常時名前ラベル: **Top 3〜5**
* 逆転点の常時全面表示は禁止

### 9.4 共通

* 画面内に「見える情報量」が多すぎる場合、必ず **削る方向** を選ぶ
* 追加要素は既存要素と描画予算を取り合うものとして扱う

---

## 10. アニメーション共通ルール

### 10.1 許可されるアニメーション

* 位置補間
* 微振動
* 外周グロー
* 選択時の軽い拡大
* 現在時刻線の更新
* フェードイン / フェードアウト

### 10.2 禁止されるアニメーション

* 常時大量パーティクル
* 背景全体の流体アニメ
* 毎更新ごとの全面再レイアウト
* 常時激しい跳ね・回転
* 意味のない装飾アニメ
* 主指標を隠すほど強い演出

### 10.3 全体トグル

全機能共通で以下を持つ。

* **Animation On**
* **Animation Off**
* **Low Load Mode**

### 10.4 Low Load Mode の挙動

Low Load Mode では以下を共通で適用する。

* 発光弱化
* 震え停止または簡略化
* 補間時間短縮
* ラベル数削減
* マーカー削減
* Canvas 解像度低下
* 表示対象件数削減

---

## 11. モバイル共通ルール

TwitCasting Livefield は、モバイルを**デスクトップの縮小版**として扱わない。
モバイルには別の制限を適用する。

### 11.1 共通方針

* 情報量を減らす
* 同時比較数を減らす
* 詳細は bottom sheet に逃がす
* 横スクロールを主導線にしない
* 低負荷モードへの移行を許容する

### 11.2 モバイル共通制限

* 常時表示ラベル数を減らす
* Top 100 のような高密度表示は禁止
* 複雑な表示モードを削る
* hover 前提のUIは禁止
* タップだけで主機能が使えることを必須とする

### 11.3 タッチ操作

* タップで選択
* ドラッグ / なぞりで時間追従
* 詳細は bottom sheet
* 閉じる / 戻るが分かりやすいこと

---

## 12. メモリ・データ保持の共通ルール

### 12.1 クライアントで保持するデータ

クライアントは **表示に必要な最小限の期間・件数のみ** を保持する。

禁止:

* 全 raw 履歴をブラウザへ読み込む
* 過去日の全配信者全時刻データを一括保持する
* 別ページ分の可視化データを先読みしすぎる

### 12.2 基本方針

* Heatmap: 最新スナップショット中心
* Day Flow: 対象日の rollup のみ
* Battle Lines: 可視対象の時系列のみ重点保持

### 12.3 解放

* 画面遷移時に不要な描画バッファ・配列を解放する
* 古いスナップショット参照を残し続けない
* タブ切替時に使っていない機能の描画ループを止める

---

## 13. 共通状態設計

全機能で以下の状態を共通に持つ。

* loading
* live
* stale
* partial
* empty
* error
* demo

### 13.1 表示原則

* 状態は隠さない
* stale / partial / demo は明確に区別する
* error 時でも、可能なら最後の正常表示を残す
* demo は本番と視覚的に明確に分ける

---

## 14. 共通UIルール

### 14.1 上部メタ

各ページは最低限以下を持つ。

* ページ名
* 短い説明
* Last updated
* Coverage
* Status

### 14.2 コントロール

各ページのコントロールは以下の原則に従う。

* 重要なものを左に寄せる
* 細かいものは減らす
* 同じ意味の切替を乱立させない
* モバイルで2行以内に収まることを優先する

### 14.3 詳細表示

* PC: 右サイドパネル
* モバイル: bottom sheet

### 14.4 色の原則

* ダーク基調
* 役割のない多色乱用は禁止
* 色は「識別」か「状態」のために使う
* 演出は色よりも光・密度・位置で表す

---

## 15. 共通禁止事項

以下は **全機能共通の禁止事項** とする。

### 15.1 描画・更新

* 毎フレーム全面再レイアウト
* 毎フレーム全件再ソート
* 毎フレーム全DOM再生成
* 大量SVGノードの主描画利用
* データ件数無制限表示
* 背景タブでの常時描画
* 表示外データの無制限保持

### 15.2 演出

* 主指標を読めなくするノイズ
* ずっと動き続ける無意味な装飾
* 低スペック端末で破綻する発光量
* 画面全体を覆う演出

### 15.3 実装

* React state に毎フレーム数値を流し続ける実装
* 描画コアのフレームワーク強依存
* モバイルでPC同等描画を強制する実装
* フェイルセーフなしの高負荷モード

---

## 16. 共通フェイルセーフ

### 16.1 原則

重いと判定された時は、**壊れる前に簡略化する**。
「高機能のまま落ちる」より、「機能を落として生き残る」を優先する。

### 16.2 自動簡略化の順序

1. 発光弱化
2. 震え停止
3. 補助マーカー削減
4. ラベル削減
5. 表示対象件数削減
6. 低解像度描画
7. 静的更新のみへ移行

### 16.3 最終手段

それでも不安定なら、以下を許可する。

* アニメーション完全停止
* 現在スナップショットのみの静的描画
* 低負荷モードをデフォルト化

---

## 17. テスト観点の共通ルール

各機能は、見た目より先に以下を満たさなければならない。

### 17.1 必須確認

* 10分以上表示してもクラッシュしない
* 連続更新中にメモリが異常増加しない
* タブ非表示→再表示で破綻しない
* モバイルで主操作が完了できる
* stale / partial / error が正しく出る
* Animation Off / Low Load Mode が有効に働く

### 17.2 不合格条件

以下のいずれかがあれば、その実装は不合格とする。

* ブラウザタブが落ちる
* スクロールやタップで明確な固まりが出る
* 数分放置でFPS低下が進行する
* ラベルやマーカーが多すぎて読めない
* モバイルで操作不能
* 描画更新でページ全体が詰まる

---

## 18. 機能間の役割分担

共通仕様として、3機能の役割重複を避ける。

### Heatmap

* 今この瞬間
* 点で読む
* 動きで熱量を見る

### Day Flow

* 1日の支配率
* 面で読む
* 総量とシェアを見る

### Battle Lines

* 個別比較
* 線で読む
* 逆転・急伸を読む

どの機能も、他機能の役割を食ってはならない。

---

## 19. 将来拡張の原則

将来、Niconico Livefield や Whowatch Livefield を追加する場合も、本書の考え方を踏襲する。

* 安定動作優先
* Canvas 主体
* D1 / KV の役割分離
* 件数上限制
* 演出は予算内のみ
* モバイル優先簡略化

---

## 20. 本仕様書の一言定義

**TwitCasting Livefield 共通仕様書は、各可視化機能がクラッシュせず、読みやすく、無料運営でも持続可能な形で成立するための最上位ルールを定める文書である。**
